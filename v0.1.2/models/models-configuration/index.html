Models configuration

This guide covers the structure of the models configuration file config.yaml. Read more about YAML here.

Model configuration header

The model configuration header consists of the following fields:

FieldDescriptionTypeRequired
nameThe name of the model, potentially including the version identifier.StringYes
descriptionA short description of the model.StringYes
authorThe author of the modelStringYes
rolesThe roles that can use the model.Array of StringsYes

A simple example of a model configuration header is shown below:

name: MyModel-v1
description: A model for predicting the weather
author: John Doe
roles:
  - user

# Other fields
# ...

Upon loading a model, the model configuration header will be validated. An error will be thrown if the model configuration header is invalid, e.g. if some of the fields are missing or are of the wrong type.
